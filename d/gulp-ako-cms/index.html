O Gulpe namiesto CMS, smradlavom PHP a geniálnych Rubystoch

2015-02-19

Problémy boli dva. Prvým bolo, že front-end developeri by síce radi písali plnohodnotné weby, ale keďže jQuery nekomunikuje s databázov a tak sa nevedeli pohnúť. Druhým bolo, že PHP síce smrdí, ale naučiť sa ho zle používať nevyžaduje veľa času a tak sa uchytilo (podobne ako Javascript).

Kedysi sme reusovali časti kódu cez iframy, až pokiaľ niekto neprišiel s include v PHP. Následný boom Wordpressu však spôsobil, že máme PHP na 75 miliónov stránok. De facto všade od jednoduchých prezentačných webovo, cez krátke formuláre odosielajúce email až po inteligetné hajzlové dosky dnes obsahuje kus PHP.

Geniálny nápad Rubystov

Geniálny nápad rozšírila najmä Ruby komunita, keď v 2008 prišli s Jekyllom. Niekto si povedal, že kvôli tagom, templatom a pagination asi nepotrebujeme obrovskú MySQL databázu plnú reklamy na Viagru, 80 eurový hosting a 75MB Wordpress zdrojákov, ale stačí nám pät skriptov.

Zrazu sa začalo zisťovať, že vlastne nepotrebujeme hosting, ale stačí Github. Že články nemusia ísť cez wysiwyg a nemusia byť uložené v databáze, ale stačí jednoduchý markdown file. Že history nemusí byť ďalšia tabuľka, ale Git repo. Že na osolanie emailu nepotrebujeme PHP script a mail server, ale vlastne iba REST call.

Generátory a 3984 Javascript alternatív

Ako tradične, keď je niečo sexy, Javascript komunita urobí asi 3984 alternatív. A tak nám vznikli assemble, blacksmith, broccoli taco, brunch, cabin, go-static, harp. heckle, hexo (zoradené abecedna, pretože všetky sú úplne rovnako nahovno).

Ďaľši problém však prišiel iba minulý rok! Totiž dnes je toľko zlých alternatív, že prakticky komunita tých 17 šikovných Javascript kóderov sa nesústreďuje na jednom mieste, ale každý sa snaží urobiť vlatný zbytočný generátor statických stránok.

Gulp ako riešenie

Riešením je začať používať tool, ktorý tu už je a funguje - Gulp a obyčajné NPM pluginy. Gulp je predsa task runner. Čo tak vytvoriť tých 5 taskov ktoré potrebujete na váš blog?

1, prekopírovať súbory do build foldra 2, konvertnúť markdown do templatov a vyprdnúť html 3, minifikovať HTML/CSS/JS a prekopírovať assety 4, streamnuť to na localhost

Done? Tomu ver. Tak funguje aj táto sajta. Neveríš? Čekni gulpfile tejto sajty ktorý ju celú generuje. Chceš vedieť ako na to? Čekni ďalší článok!
